STREAMING mode — when total GPU requested ≤ available GPUs, all stages run simultaneously (pipeline parallelism). Our standard configuration requests ~1.5–2.5 GPU total; using 3 GPUs unlocks streaming and gives a ~40% wall-clock speedup over single-GPU BATCH mode.
